# 性能优化报告（简版）

## 资源体积

- 机器人模型：`public/models/robot.glb` 约 453 KB
- 生产构建产物（gzip）：JS 约 323 KB（不含浏览器缓存与运行时纹理/几何占用）

## 加载与交互优化

- 使用 `@react-three/drei` 的 `useGLTF` + `useGLTF.preload()` 预加载模型，减少首屏等待。
- 动画切换采用 `fadeIn/fadeOut`，避免硬切造成的卡顿与视觉跳变。
- 每帧更新只做必要的位移/旋转、重力与地形射线检测，以及 morph 权重插值，逻辑保持轻量。

## 渲染与 Draw Call

- 场景结构保持简单：地面、网格、天空、少量盒体平台 + 单个角色模型。
- 如需进一步降低 draw call：
  - 通过 DCC 合并静态几何（平台/装饰）或使用实例化（Instancing）。
  - 避免大量独立材质，尽量复用材质。

## 建议（如果要进一步压缩首屏）

- 对 GLB 使用 Draco 或 Meshopt 压缩（需要配套解码器与加载配置）。
- 开启贴图压缩（KTX2/BasisU）并限制贴图分辨率。
- 若后续引入更多功能，建议对非首屏模块做按需加载（动态 import）。
