# 性能优化报告（简版）

## 资源体积

- 机器人模型：public/models/robot.glb 约 453 KB
- 生产构建产物（gzip）：JS 约 323 KB（不含浏览器缓存与运行时纹理/几何占用）

---

## 加载与交互优化

- 使用 @react-three/drei 的 useGLTF + useGLTF.preload() 进行模型预加载，减少首屏加载时间。
- 动画切换采用 fadeIn / fadeOut，实现平滑过渡，避免硬切带来的卡顿与视觉跳变。
- 每帧更新仅执行必要逻辑，包括：
  - 位移与旋转更新
  - 重力与地形检测
  - morph 权重插值

整体保持计算开销轻量。

---

## 渲染与 Draw Call 控制

当前场景结构较为简单：

- 地面 + 网格 + 天空
- 少量静态平台
- 单个角色模型

优化策略：

- 控制场景复杂度，避免过多独立对象
- 优先复用材质，减少材质切换
- 可通过实例化（Instancing）或模型合并降低 draw call

---

## 后续优化建议

如需进一步优化性能，可考虑：

- 使用 Draco 或 Meshopt 压缩 GLB 模型
- 使用 KTX2 / BasisU 压缩贴图资源
- 限制贴图分辨率
- 对非核心模块进行按需加载（动态 import）
